# Prompt для Codex: собрать Android-проект в GitHub под Android Studio

Нужно создать **полноценный Android Studio проект на Kotlin + Jetpack Compose** в текущем GitHub-репозитории.

## Контекст работы

Я буду работать так:

1. Ты, Codex, вносишь изменения и пушишь их в GitHub-репозиторий.
2. Я со своего GitHub потом открываю этот проект в **Android Studio**.
3. Поэтому проект должен быть собран как нормальный **Android Studio Gradle project**, без экзотики, без ручных костылей, без нестандартной среды сборки.

## Общая задача

Собери MVP Android-приложения **«Приватная сессия VPN»** по приложенному ТЗ.

Главная цель приложения:

1. Работать как обычный VPN-клиент.
2. Поддерживать режим **«Приватная сессия»**, где только доверенные приложения получают сеть через VPN.
3. Все остальные приложения должны оставаться без сети, если пользователь включил Android-настройку **«Блокировать подключения без VPN»**.
4. Поддерживать **Xray / VLESS / REALITY** как главный приоритет.
5. Поддерживать импорт как строк `vless://...`, так и **raw Xray JSON** вроде desktop-конфигов.
6. Иметь встроенную **лабораторию сканера VPN**, чтобы можно было проверять практические утечки.

## Очень важные ограничения

1. **Не использовать root.**
2. **Не использовать Shizuku.**
3. **Не требовать ADB, shell-команд и ручных внешних действий.**
4. Всё должно работать **из коробки** в рамках возможностей обычного Android-приложения.
5. Весь пользовательский интерфейс должен быть **на русском языке**.
6. Не надо обещать полную невидимость VPN на уровне всей системы.
7. Нужно делать честный продукт: Xray/VLESS/REALITY клиент + приватная сессия + self-check.

## Что нужно реализовать в MVP

### 1. Базовый каркас приложения

Сделай:

1. экран запуска
2. экран первого запуска
3. главный экран
4. экран профилей
5. экран добавления профиля
6. экран списка приложений
7. экран настройки приватной сессии
8. экран активной сессии
9. экран проверки безопасности
10. экран лаборатории сканера VPN
11. экран журналов
12. экран настроек DNS
13. экран настроек приложения

Все названия, кнопки, статусы и сообщения в UI должны быть на русском.

### 2. Импорт конфигов

Нужно поддержать:

1. импорт `vless://...`
2. импорт `vmess://...`
3. импорт `trojan://...`
4. импорт из буфера обмена
5. импорт из файла
6. импорт raw Xray JSON

Особенно важно: пользователь может вставить JSON-конфиг такого типа, как desktop Xray-конфиг:

- `inbounds`
- `outbounds`
- `log`
- `routing`
- `dns`

Например inbound с `socks`, outbound с `vless`, `streamSettings.security = reality`.

Нужно сделать нормализатор импорта:

1. парсить конфиг
2. выделять поддерживаемые части
3. не падать на неизвестных секциях
4. не логировать секреты
5. аккуратно показывать пользователю, что импортировано полностью, а что частично

### 3. Подключение по VPN

Нужно реализовать реальное подключение через `VpnService`.

Приоритет №1: **VLESS + REALITY**.

Codex должен выбрать движок или способ интеграции, который реально подходит для Android и Xray-совместимых конфигов.

Важно:

1. не использовать `allowBypass()`
2. явно задавать DNS внутри туннеля
3. сделать lifecycle устойчивым к старту/остановке
4. не превращать проект в экспериментальную свалку

### 4. Режим «Приватная сессия»

Это ключевая часть.

Нужно реализовать:

1. список доверенных приложений
2. сбор allowlist через `VpnService.Builder.addAllowedApplication()`
3. отдельную кнопку **«Запустить приватную сессию»**
4. проверку системных Android-настроек:
   - `Всегда включённый VPN`
   - `Блокировать подключения без VPN`
5. понятный экран-инструкцию, если пользователь не включил эти настройки

Важный смысл:

- доверенные приложения должны иметь сеть через VPN
- недоверенные приложения должны остаться без сети, если включён системный kill-switch Android

### 5. Локальный SOCKS

Мы не запрещаем SOCKS.

Но по умолчанию локальный SOCKS должен быть выключен.

Если пользователь включает локальный SOCKS, реализуй **безопасный режим**:

1. только `127.0.0.1`
2. логин/пароль
3. по умолчанию случайный порт
4. по умолчанию случайные учётные данные на каждую сессию
5. явная маркировка в UI: безопасный / небезопасный

Нужна проверка в модуле безопасности:

1. открыт ли локальный SOCKS
2. требует ли авторизацию
3. доступен ли анонимно

### 6. Leak Guard

Нужен модуль `LeakGuard`, который перед запуском сессии проверяет:

1. не включён ли bypass
2. настроен ли DNS внутри туннеля
3. нет ли опасного локального inbound
4. корректно ли сформирован allowlist
5. не включён ли небезопасный SOCKS без предупреждения

### 7. Лаборатория сканера VPN

Нужно сделать встроенный экран **«Лаборатория сканера VPN»**.

Он должен эмулировать обычное приложение, которое пытается понять, есть ли VPN.

Минимальный функционал:

1. чтение активной сети через `ConnectivityManager`
2. чтение `NetworkCapabilities`
3. чтение `LinkProperties`
4. запрос на backend для определения внешнего IP
5. показ страны и ASN
6. проверка localhost SOCKS
7. визуальное сравнение сценариев:
   - обычный VPN
   - split tunneling
   - приватная сессия

### 8. Test App / Scanner App

Если успеваешь в рамках MVP, сделай отдельный модуль или flavor простого test-app, который:

1. показывает сетевое состояние через Android API
2. делает запросы на `/ip`, `/geo`, `/asn`
3. пытается проверить localhost SOCKS
4. показывает понятный отчёт

Если не успеваешь сделать отдельный модуль сразу, хотя бы заложи архитектурно и опиши TODO в документации.

### 9. Тестовый backend

Если не хочется тратить много времени на backend, можно сделать простой шаблон сервиса или документированный mock-контракт.

Нужные endpoints:

1. `/ip`
2. `/geo`
3. `/asn`
4. `/blacklist-check`
5. `/echo`

Минимум — подготовить интерфейс клиента и моковые ответы.

## Очень важные UX-требования

1. Весь UI на русском.
2. Не перегружать главный экран.
3. Основная кнопка: **«Подключить»** или **«Запустить приватную сессию»**.
4. Если системные настройки Android не включены, приложение должно не просто ругаться, а вести пользователя по понятному сценарию.
5. Интерфейс должен выглядеть как нормальный Android-клиент, а не как лаборатория для инженеров.
